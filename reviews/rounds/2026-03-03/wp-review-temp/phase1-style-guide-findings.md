## Notes on Scope

The following areas were examined and found to be in compliance or otherwise noted:

**Passing checks:**
- Sections 1–2 were not reviewed per AGENTS.md constraint ("Do NOT review or flag these sections").
- Vulnerability communication templates (§7.1): Both the Authenticated and Unauthenticated templates are present, complete, and internally consistent in format and field coverage.
- Two-font system (§6.1–6.3): Correctly defined with clear boundary rules. The `wp-cli` example in §6.2 (Finding 6) is the only detected self-compliance failure in this system.
- Section structure: §§3–7 (objective guidance), §8 (glossary), §9 (operational workflow appendix) are all present and correctly organized. §10 (References and Further Reading) is an unlisted but well-formed addition.
- Code block integrity: The Style Guide contains no fenced code blocks — consistent with AGENTS.md §4 which specifies "Code examples: None" for this document. No code block escaping issues are applicable.
- Glossary cross-references pointing to non-existent terms: All cross-references in italic format were checked. Every italicized target exists as a glossary entry. The monospace cross-reference errors in Findings 3–5 are formatting errors, not broken links.

**Additional instance of Finding 4/5 class (not separately numbered):**
The `wp-admin` entry (line 573) contains `See also: *Dashboard*, \`wp-login.php\`.` — `wp-login.php` is a glossary entry and should use italic format `*wp-login.php*` rather than monospace backticks. This is a fourth instance of the same cross-reference formatting error class documented in Findings 4 and 5. It is noted here rather than as a separate finding because it is identical in cause and fix.

**LEMP — does not meet the 2-document threshold:**
The term LEMP appears only in the Runbook (§3.1 heading, §3.2 table, and Runbook Glossary). It is absent from both the Benchmark and the Hardening Guide (verified by exact-string search). It does not qualify for a mandatory glossary entry under the AGENTS.md constraint ("every technical term used in 2+ of the other documents"). No action required unless LEMP is added to a second document.

**Alphabetical ordering rationale for Findings 1 and 2:**
Finding 1 (`FORCE_SSL_ADMIN` vs `FUD`): Alphabetical comparison of the initial characters after common prefix 'F': "FO" < "FU" because 'O' (ASCII 79) precedes 'U' (ASCII 85). Therefore `FORCE_SSL_ADMIN` must appear before `FUD`.
Finding 2 (`mu-plugin` vs `Multisite`): Case-insensitive comparison of characters after common prefix 'mu': '-' (ASCII 45) precedes 'l' (ASCII 108). Therefore `mu-plugin` must appear before `Multisite`.
