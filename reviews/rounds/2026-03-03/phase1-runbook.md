# Phase 1 — Runbook Audit (@RunbookAgent, Sonnet)

Date: 2026-03-03

## Summary

Audit covered all sections of the WordPress Operations Runbook (v2.0, 3027 lines). Findings are grouped by severity. Total: 3 Critical, 6 High, 7 Medium, 4 Low.

- **Critical (3):** Nonexistent WP-CLI flags used in runnable commands that will fail on execution.
- **High (6):** Incorrect WP-CLI subcommands, a broken/fictitious procedure, a misleading database rollback command, and redundant Nginx directives that violate the Style Guide code block rules.
- **Medium (7):** Inconsistent plugin-dependent command variants, a wrong code fence language tag, misleading wp-config.php SMTP constants, and glossary gaps.
- **Low (4):** Minor annotation gaps, a version claim that cannot be verified, and minor procedural observations.

| 7 | Section 5.4, lines 585–588 and Section 6.6, lines 1038–1041 | Both Nginx location blocks for `xmlrpc.php` and `wp-cron.php` include `deny all;` followed by `return 403;`. The `return 403;` after `deny all;` is unreachable dead code in Nginx. | High | Remove `return 403;` from both location blocks. The corrected xmlrpc.php block: `location = /xmlrpc.php { deny all; }` The corrected wp-cron.php block: `location = /wp-cron.php { deny all; }` | Style Guide §2 (Code Blocks); Nginx documentation on `deny`. |

| 10 | Section 6.7, lines 1059–1063 | The `wp option update mailer`, `wp option update mailer_host`, `wp option update mailer_port`, `wp option update mailer_encryption`, and `wp option update mailer_username` commands are plugin-specific option keys for the WP Mail SMTP plugin. They are not WordPress core option names. | 11 | Section 10.2, lines 1849–1851 | `wp plugin list --status=active \| grep -E "activation\|installed"` — the `grep -E` with `\|` inside a code block is syntactically correct ERE (a literal `\|` in ERE means a literal pipe, not alternation; alternation in ERE is bare `|`). This is actually the same bug pattern warned against in AGENTS.md §2: inside a `bash` fenced code block, the pipe is already escaped. In ERE mode (`grep -E`), `\|` is treated as a literal pipe character, not alternation. The pattern will never match typical `wp plugin list` output. The intent appears to be matching columns containing "active", but the pattern strings "activation" and "installed" do not match the actual `--status=active` column values ("active", "inactive", "must-use", "dropin"). | Medium | The grep is logically broken regardless of escape. | 12 | Section 6.4, line 879 | `SELECT COUNT(*) AS old_slug_count FROM $(wp db prefix)postmeta WHERE meta_key = '_wp_old_slug'` — the section header says "Check revision count" but this query counts old post slugs (entries with `_wp_old_slug` meta key), not revisions. Revisions are stored as posts in `wp_posts` with `post_type = 'revision'`. | Medium | Change the comment to "Check old slug count" and add a separate, correct revision count query: `wp db query "SELECT COUNT(*) FROM $(wp db prefix)posts WHERE post_type = 'revision';"` |

| 17 | Appendix A.2, lines 2576–2581 | The permission reset script sets `chmod 600 "$WP_ROOT/../backup"/*` (600 = owner read/write only for files). This is reasonable for backup files. The more useful first command in a broken-site scenario is checking logs (`tail -30 /var/log/php-errors.log`) or `wp plugin list --skip-plugins --skip-themes`. | Low | Consider changing the first command for "Site returns 500 error" to `tail -30 /var/log/php-errors.log` or `wp plugin list --skip-plugins --skip-themes --format=table` to provide a command that is more likely to succeed when WordPress cannot bootstrap. |

## Glossary Gap Summary (for Style Guide cross-check)

The following terms appear in the Runbook but have no entry in the Runbook's own glossary and, upon review of the Style Guide glossary (Section 8), also appear to be absent or underspecified there:

- `2FA` / Two-Factor Authentication
- `AIDE` (Advanced Intrusion Detection Environment)
- `CSP` (Content Security Policy)
- `HSTS` (HTTP Strict Transport Security)
- `UFW` (Uncomplicated Firewall)
- `FIM` (File Integrity Monitoring) — present in Runbook glossary but not verified in Style Guide
- `LEMP` — present in Runbook glossary but spelled out only partially (`Linux, Nginx, MySQL, PHP`); `E` expansion missing from glossary entry (it stands for `Engine X` / Nginx, rendered as "E" for historical/phonetic reasons)

The Style Guide glossary should be reviewed for these terms as part of the cross-document alignment pass.
